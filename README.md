> **AI-система автоматического аудита и улучшения ТЗ научных проектов**
> Хакатон НТЦ LAB | 14–15 апреля 2025 | Команда **BitUnion**

---

## Что это такое

Вы загружаете Техническое Задание (DOCX или PDF), система за 30 секунд:

1. Читает документ и проверяет структуру по официальному шаблону МОН РК
2. Оценивает ТЗ по 7 критериям — точно по листу жюри (`Оценка_ТЗ_шаблон.xlsx`)
3. Объясняет каждую оценку и выдаёт список критических ошибок
4. Генерирует улучшенный вариант слабых разделов ТЗ — можно скачать

## Как запустить

### Требования

- Python 3.11+
- Node.js 18+
- API-ключ Google Gemini (или другой LLM — см. ниже)

### Шаг 1 — Backend

```bash
cd backend

# Создать виртуальное окружение
python -m venv venv

# Активировать (Windows)
venv\Scripts\activate

# Установить зависимости
pip install -r requirements.txt

# Скопировать конфиг и добавить ключ
copy .env.example .env
# Откройте .env и вставьте свой GOOGLE_API_KEY

# Запустить сервер
uvicorn main:app --reload --port 8000
```

После запуска: `http://localhost:8000/docs` — интерактивная документация API.

### Шаг 2 — Frontend

```bash
cd frontend

# Установить пакеты
npm install

# Запустить в режиме разработки
npm run dev
```

После запуска откройте `http://localhost:5173` в браузере.

### Быстрый старт через Docker

```bash
# В корне проекта
docker-compose up --build
```

Frontend: `http://localhost:5173` | Backend: `http://localhost:8000`

## Настройка LLM-провайдера (файл `backend/.env`)

По умолчанию используется Google Gemini. Переключить на другой провайдер — одна строчка:

```env
# Провайдер: gemini | openai | anthropic | ollama
LLM_PROVIDER=gemini

# Google Gemini (получить ключ: https://aistudio.google.com)
GOOGLE_API_KEY=ваш_ключ_здесь
GOOGLE_MODEL=gemini-1.5-pro

# OpenAI GPT-4o (если LLM_PROVIDER=openai)
OPENAI_API_KEY=ваш_ключ_здесь

# Anthropic Claude (если LLM_PROVIDER=anthropic)
ANTHROPIC_API_KEY=ваш_ключ_здесь

# Ollama — локально, без интернета (если LLM_PROVIDER=ollama)
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=qwen2.5:14b
```

Для Ollama нужно отдельно установить и запустить `ollama run qwen2.5:14b`.

## Как работает система

### Общий поток

```
Пользователь загружает файл
        ↓
[document_parser.py]
  Парсит DOCX/PDF → чистый Markdown
  Проверяет 5 обязательных разделов шаблона МОН РК
  Добавляет структурный отчёт ✅/❌ в начало текста
        ↓
[ai_scorer.py]
  Передаёт текст + системный промпт в LLM
  Системный промпт содержит:
    - 7 официальных критериев с весами из xlsx
    - Few-Shot пример (реальное ТЗ «Цифровой полигон»)
    - Правила штрафов за расплывчатые формулировки
  LLM возвращает строгий JSON
        ↓
[Pydantic TZAuditResult]
  Валидирует JSON
  Пересчитывает total_score (защита от галлюцинаций)
  Автоматически выставляет букву A/B/C/D/F
        ↓
[React Dashboard]
  Спидометр итогового балла
  Радарная диаграмма по 7 критериям
  Вкладки: Оценки / Критические ошибки / Улучшенное ТЗ
  Кнопка скачать улучшенный вариант (.md)
```

### 7 критериев оценки

| Критерий | Вес | На что смотрим |
|---|---|---|
| Стратегическая релевантность | 20% | Конкретные пункты ГПИИР, Стратегии 2050, НПРаз |
| Цель и задачи | 10% | Чёткость, измеримость, логика от задач к цели |
| Научная новизна | 15% | Наличие «впервые», отличие от мирового уровня |
| Практическая применимость | 20% | Реальные партнёры, соглашения, пилоты |
| Ожидаемые результаты | 15% | KPI с числами: публикации Q1/Q2, патенты, прототипы |
| Соц-экономический эффект | 10% | Рабочие места, % экономии — всё в цифрах |
| Реализуемость | 10% | CV команды, оборудование, бюджет по годам |

Итоговый балл = `Score1×0.2 + Score2×0.1 + Score3×0.15 + Score4×0.2 + Score5×0.15 + Score6×0.1 + Score7×0.1`

### Что делает `check_tz_structure`

Прямо перед отправкой в LLM система проверяет наличие 5 обязательных разделов по официальному шаблону МОН РК:

- Раздел 1: Общие сведения (приоритет, направление)
- Раздел 2: Цели и задачи программы
- Раздел 3: Стратегические и программные документы (с конкретными пунктами)
- Раздел 4: Ожидаемые результаты с KPI (4.1 + 4.2)
- Раздел 5: Бюджет по годам

Результат добавляется в начало текста. ИИ сразу видит, каких разделов не хватает — это повышает точность оценки.

## Граничные оценки

| Балл | Оценка | Значение |
|---|---|---|
| 85–100 | **A** | Отлично, высокий шанс одобрения |
| 70–84 | **B** | Хорошо, мелкие доработки |
| 55–69 | **C** | Удовлетворительно, нужны правки |
| 40–54 | **D** | Слабо, требует серьёзной переработки |
| < 40 | **F** | Критические недостатки |
